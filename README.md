# NEO GUARD REEL POUCH — 商品ランディングページ

釣具ECサイトを想定して制作した架空商品「**NEO GUARD REEL POUCH**」（ネオプレーン素材のリールポーチ）の商品ランディングページです。

- 公開URL: <https://neoguard.7g3.net/>
- 制作: **Nagisa Dozono** — <https://7g3.net/>

> ⚠️ **本ページは、釣具ECサイトを想定して制作したポートフォリオ作品です。**
> 商品・ブランドは架空のもので、実在の商品・ブランドとは関係がなく、実際の販売・決済は行いません。

採用応募用ポートフォリオとして、構成力・購入導線の設計・実装力・アクセシビリティ・パフォーマンス配慮を1つのページで示すことを目的にしています。商品ビジュアルも含めて完成した状態です。

## 商品ビジュアルについて

商品画像・図解はすべて、著作権の問題がないよう **SVGで制作したオリジナルの商品モックアップ** です（実写真・外部素材は使用していません）。ネオプレーン製リールポーチを題材に、スピニングリール・ロッド・スプール・固定ベルト・縫い目・素材の厚みなどを表現しています。サイトカラー（ネイビー基調）で統一し、以下を制作しました。

- ヒーローのメインビジュアル（ロッドに付けたリールへポーチを装着した状態）
- 商品特徴3点（クッション構造 / 撥水 / 装着）
- Before / After（バッグ内での接触 → ポーチで個別保護）
- 使い方3ステップ（かぶせる → ベルトを巻く → 面ファスナーで固定）
- OGP画像（1200×630, `public/ogp.png`）

`public/images/` 配下に個別のSVGとして置いているため、将来実写真（WebP / AVIF）へ差し替える場合も、同じ `src` を置き換えるだけで対応できます。

## 主な機能

- **サイズ診断（Size Finder）** — 2問に答えるとおすすめサイズを提示。クライアントサイドのみで動作し、結果から購入エリアへスクロール＆選択サイズを自動反映。ステップ切り替え時のフォーカス移動・スクリーンリーダー向けのステップ通知に対応。
- **商品購入エリア** — カラー / サイズ / 数量の選択、選択中サマリー（例: `Black / Mサイズ / 数量1`）、在庫・配送・返品情報。カラー・サイズはボタン式で、選択状態を「枠線・チェックマーク・aria」で表現（色のみに依存しない）。
- **カート追加はデモ動作** — 決済は行わず、選択内容を含むメッセージを `aria-live` で通知。
- **サイズ比較表** — デスクトップは表、モバイルはカード型に切り替え（横スクロールなし）。
- **FAQ アコーディオン** — `button` + `aria-expanded` / `aria-controls` による開閉。
- **モバイルメニュー** — フォーカストラップ、開放時に先頭項目へフォーカス、Escや項目選択で閉じてボタンへ復帰、背景スクロール制御。
- **モバイル固定CTA** — ファーストビュー通過後に表示、購入エリア付近では非表示。`env(safe-area-inset-bottom)` 対応。
- **簡易アナリティクス** — 主要イベントを1つの関数に集約（開発時は console 出力、本番は GA4 等に差し替え可能。個人情報は記録しない）。

## 設計上の意図

### デザイン
- **モバイルファースト**で、商品ビジュアルを主役に据えた清潔感のあるレイアウト。
- カラーパレットはネイビー基調＋CTAにオレンジ（`#E45A2A`）を採用し、購入ボタンの視認性を最優先。
- 過度なグラデーション・角丸・ガラスモーフィズム・意味のない装飾を排除し、実在ECサイトに近い現実的なトーンに。
- アニメーションは 150〜300ms 程度に限定し、`prefers-reduced-motion` を尊重。

### ECサイトを意識した購入導線
- ファーストビューに商品名・キャッチコピー・価格・送料条件・評価・CTAを集約し、離脱前に主要情報が伝わる構成。
- 「悩み → 特徴 → Before/After → サイズ診断 → 比較表 → 使い方 → 購入 → レビュー → FAQ → 最終CTA」と、疑問を解消しながら購入へ向かう順序。
- サイズ診断・最終CTA・固定CTA・ヘッダーの購入ボタンなど、複数の導線から購入エリアへ到達可能。

### アクセシビリティ
- セマンティックHTML、スキップリンク、1ページ1つの `h1` を含む見出し構造。
- キーボード操作と明確なフォーカスリング。選択状態は色のみに依存しない。
- モバイルメニューは `aria-expanded` / `aria-controls`＋フォーカストラップ、開放中は背景スクロールを制御、Escで閉じる。
- フォーム部品にラベル、通知は `aria-live`、画像には内容を説明する `alt`。

### パフォーマンス
- すべての画像に `width` / `height` を指定してレイアウトシフトを抑制。
- ファーストビュー以外の画像は `loading="lazy"`。
- 画像は軽量なSVG（将来 WebP / AVIF への差し替えも容易な構造）。
- 追加ライブラリを増やさず、JSを最小限に。

### SEO・構造化データ
- 日本語の `title` / `meta description`、OGP（`canonical` / `og:url` は `https://neoguard.7g3.net/`）。
- 架空商品を実在の販売ページと誤認されないよう、構造化データは **Product ではなく `CreativeWork`** を採用。評価（`aggregateRating`）・在庫・販売情報（`offers` / `availability`）は含めず、「ポートフォリオ作品」であること、制作者・制作年・使用技術（keywords）を記述。
- レビューは画面設計を示すための架空データであることをページ上に明記し、検索エンジン向けの評価データ（構造化データ）としては一切出力しない。

## ディレクトリ構成

```
src/
├─ components/        各セクション・UIコンポーネント（1コンポーネント1フォルダ + CSS Module）
├─ data/              商品・特徴・FAQ・レビューのデータ
├─ lib/               analytics / sizeFinder ロジック / scroll ヘルパー
├─ types/             型定義
├─ styles/            グローバルCSS（デザイントークン）
├─ constants.ts       セクションID
├─ App.tsx            ページ全体の組み立て・共有state
└─ main.tsx           エントリーポイント
public/images/        オリジナルの商品モックアップSVG
public/ogp.png        OGP画像（1200×630）
```
